# 网页版日本麻将游戏设计文档

## 概述

单人离线网页版日本麻将游戏，玩家与三个电脑对手对战。采用标准日麻规则（雀魂风格），困难级AI。随机种子由用户按确定时的 Linux 时间戳决定。技术栈为 Vite + React，多文件模块化架构，可部署到 GitHub Pages。

## 技术决策

| 项目 | 选择 | 原因 |
|------|------|------|
| 规则 | 标准日麻 | 雀魂风格，最广泛使用的规则 |
| AI难度 | 困难（高手级） | 向听数计算+攻防转换+安全牌判断 |
| 技术栈 | Vite + React | 开发快、构建小、GitHub Pages友好 |
| 架构 | 多文件模块化 | 日麻规则复杂，需职责分离便于维护 |
| 牌面 | 传统日麻风格（CSS渲染） | 无图片依赖，纯离线可用 |
| 随机种子 | 用户确认时Linux时间戳 | 可重现的游戏序列 |

## 游戏流程

### 完整流程

1. **开始界面** → 显示当前时间戳作为种子，用户点击确认开始
2. **洗牌配牌** → 用 xorshift128+ 种子洗牌，每人发13张，庄家14张
3. **自摸阶段** → 庄家先自摸→判断是否天和→否则出牌
4. **出牌循环** → 依次：出牌 → 其他玩家可吃/碰/杠/和 → 若无操作则下一家自摸 → 循环
5. **和牌/流局** → 和牌时结算役与点数，流局时判断听牌/noten
6. **局数推进** → 和牌/流局后推进局数，连续直到南4局结束
7. **终局结算** → 显示最终排名和点数

### 状态结构

```js
gameState = {

```

### 关键交互流程

- **玩家出牌**：点击手牌中的一张 → 牌打出 → 进入其他玩家反应阶段
- **AI出牌**：自动决策 → 有短暂动画延迟（0.3-0.8秒）→ 出牌
- **吃碰杠选择**：出牌后如果多家可操作，按优先级排序（和 > 杠 > 碰 > 吃），玩家需要手动确认，AI自动决策
- **立直**：玩家手牌满足条件时，操作面板显示立直按钮；立直后只能打自摸牌

## 困难 AI 策略

### 手牌评估器（evaluator.js）

- **向听数计算**：核心算法，精确计算当前手牌到和牌的最少步骤数
- **有效牌分析**：列出所有能降低向听数的牌，以及它们的出现概率（基于已见牌数）
- **打牌评分**：对每张可打的牌计算综合评分 = 降向听数权重 × 出现概率权重 × 攻防权重

### 核心策略（strategy.js）

AI 每次决策的流程：

```
自摸牌后：
  1. 计算当前向听数
  2. 如果向听数=0 → 检查是否能和牌 → 和了
  3. 如果向听数=1 → 判断是否立直（考虑手牌强度、巡目、点数情况）
  4. 对所有可打牌评分 → 打评分最低的牌

打牌评分因素：
  - 保留有效牌组合的潜力
  - 避免打出对手可能需要的牌（攻防转换）
  - 考虑dora（宝牌）关联牌的价值加成
  - 后巡时偏向立直而非默听
```

### 反应决策（reaction.js）

对其他玩家出牌的反应：

- **和牌**：能和就接（除非振听）
- **碰/杠**：向听数 ≤ 2 且碰后手牌有明确方向时碰；向听数 ≤ 1 时考虑明杠
- **吃**：只在向听数 ≤ 2 且吃后能大幅改善牌型时吃，否则跳过
- **防守意识**：对手立直后，切换防守模式——优先打安全牌（现物、筋牌），宁可提高向听数也要避免放铳

### AI 行为节奏

- AI 打牌/决策有 0.3-0.8 秒的随机延迟，模拟思考时间，让玩家能观察到游戏进程
- 不做"完美"决策——困难 AI 也有微小的随机扰动，避免行为完全可预测

## UI 与视觉设计

### 布局结构

采用经典日麻游戏的四方位布局：

```
        ┌─────────────┐
        │  电脑2（对面）│
        │  手牌(横向背面) │
        │  出牌区      │
        └─────────────┤
┌───────┤  牌山/宝牌   ├───────┐
│电脑3  │  中央信息区  │电脑1  │
│(上家) │            │(下家) │
│手牌   │            │手牌   │
│(竖向) │            │(竖向) │
└───────┤            ├───────┘
        ├─────────────┤
        │  玩家（自己）│
        │  手牌(横向正面)│
        │  出牌区      │
        └─────────────┘
```

- 玩家手牌在最下方，正面显示，方便操作
- AI 手牌在各自方位，默认背面显示（和牌/吃碰杠后的明牌正面显示）
- 中央区域显示牌山剩余数、宝牌、本场/局数信息

### 牌面渲染

传统日麻风格，用 CSS + Unicode 字符绘制，无需图片：

- **万子**：一~九的汉字 + 红色"萬"字
- **筒子**：圆形图案表示数字（用 CSS 圆点排列）
- **索子**：竹节图案表示数字
- **风牌**：東南西北 汉字
- **三元牌**：白（空白框）、發（绿色）、中（红色）
- **赤牌**：赤五万(红字)、赤五筒(红圆)、赤五索(红竹节) 有特殊红色标记

每张牌约 40×56px，白色底、黑框、内部字符/图案。

### 操作面板

当玩家有可选操作时，在屏幕下方弹出操作面板：

- **吃**：显示可选的组合（如左吃/中吃/右吃），点击选择
- **碰**：碰按钮，点击确认
- **杠**：明杠/加杠按钮（如有两者可选）
- **立直**：立直按钮，点击后手牌锁定，只能打自摸牌
- **和**：自摸和/荣和按钮
- **跳过**：不执行任何操作，继续游戏

### 动画与反馈

- 出牌：牌从手中飞到出牌区，约 0.2 秒
- 吃碰杠：被吃的牌飞到操作者面前，约 0.3 秒
- 和牌：手牌翻开展示，役名和点数弹出显示
- 立直：立直棒（1000点棒）飞到中央

## 规则细节与和牌判定

### 基本规则

- 136张牌（万1-9×4、筒1-9×4、索1-9×4、风4种×4、三元3种×4）
- 含赤牌：赤五万×1、赤五筒×1、赤五索×1（各替换1张普通五，共3张赤牌）
- 庄家轮换：和牌时庄家继续，流局时若庄家听牌继续否则轮换
- 起始点数：25000点
- 游戏长度：东风战（东1-4局）→ 南风战（南1-4局）→ 终局

### 立直规则

- 条件：门前清（无吃碰杠）、向听数≤1、点数≥1000
- 立直后不可换手牌，只能打自摸牌
- 双立直：第一巡就立直
- 暗杠和加杠在立直状态下仍可执行（不影响门前清）

### 振听规则

- 同巡振听：自己曾打出的牌不能用于荣和
- 立直振听：立直后见逃（不和）则永久振听
- 牌山振听：其他玩家打出过你需要的牌但你没和

### 和牌判定（validator.js）

三种和牌形式：

- 基本型：4个面子 + 1个雀头（14张）
- 七对子：7组对子（14张）
- 国士无双：13种幺九各1张 + 任意1张幺九（14张）

### 役列表

**1番**：立直、门前清自摸和、一发、断幺九、平和、一杯口、役牌（自风/场风/三元）、岭上开花、抢杠、海底摸鱼/河底捞鱼

**2番**：双立直、七对子、混全带幺、一气通贯、三色同顺、三色同刻、对对和、三暗刻、小三元

**3番**：混一色、纯全带幺、二杯口

**6番**：清一色

**役满**：天和/地和、国士无双、四暗刻、大三元、字一色、小四喜、大四喜、绿一色、清老头、九莲宝灯

### 计点规则

- 基本点 = 符 × 2^(番数+2)
- 最低保证：30符1番 → 1000点，20符2番 → 1500点
- 役满固定32000点（子家）、48000点（庄家）

### 流局处理

- 荒牌流局：牌山耗尽无人和牌 → 检查是否有人听牌
- 听牌/noten 罚：全noten时庄家+3000/子家-1500；一人听牌+3000/其余-1000；两人听牌+1500/noten-1500
- 四家立直流局：四家均立直时强制流局，本场+1，庄家继续
- 四风连打流局：东1局四家第一巡都打出相同风牌时强制流局
